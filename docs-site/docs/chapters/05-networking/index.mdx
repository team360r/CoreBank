# Chapter 5: Talking to the Tower

> *"A pilot who can't talk to the tower is flying blind. Communication is survival."*
> — Flight training manual

**Estimated time:** ~25 minutes | **Focus:** Networking & Async | **Branch:** `chapter-5-networking`

So far, FlightBank's data has been hardcoded. Real banking apps fetch account balances and transactions from a server. In this chapter you will add HTTP networking, parse JSON into your model classes, handle loading and error states, and wire everything together with `FutureBuilder`.

## 1. The http Package

Flutter does not include an HTTP client in the core SDK. The first-party `http` package provides one.

### Step 1: Add the dependency

```bash
flutter pub add http
```



### Step 2: Make a basic GET request

```dart title="lib/services/api_service.dart"
import 'dart:convert';
import 'package:http/http.dart' as http;

class ApiService {
  static const _baseUrl = 'https://api.flightbank.dev/v1';

  final http.Client _client;

  ApiService({http.Client? client}) : _client = client ?? http.Client();

  Future<List<dynamic>> _get(String path) async {
    final uri = Uri.parse('$_baseUrl$path');
    final response = await _client.get(uri, headers: {
      'Content-Type': 'application/json',
      'Accept': 'application/json',
    });

    if (response.statusCode != 200) {
      throw ApiException(response.statusCode, response.body);
    }

    return jsonDecode(response.body) as List<dynamic>;
  }
}

class ApiException implements Exception {
  final int statusCode;
  final String message;

  const ApiException(this.statusCode, this.message);

  @override
  String toString() => 'ApiException($statusCode): $message';
}
```



:::tip[WHY THIS MATTERS]
Injecting `http.Client` through the constructor makes `ApiService` testable. In tests you will pass a `MockClient` that returns canned JSON responses without hitting a real server.

:::

## 2. JSON Parsing

Dart does not generate JSON code for you out of the box. You write `fromJson` factory constructors and `toJson` methods on your model classes. (Later, packages like `json_serializable` can automate this.)

```dart title="lib/models/account.dart"
class Account {
  final String id;
  final String name;
  final String number;
  final double balance;

  const Account({
    required this.id,
    required this.name,
    required this.number,
    required this.balance,
  });

  factory Account.fromJson(Map<String, dynamic> json) {
    return Account(
      id: json['id'] as String,
      name: json['name'] as String,
      number: json['number'] as String,
      balance: (json['balance'] as num).toDouble(),
    );
  }

  Map<String, dynamic> toJson() => {
        'id': id,
        'name': name,
        'number': number,
        'balance': balance,
      };

  String get maskedNumber => '****${number.substring(number.length - 4)}';

  String get formattedBalance {
    final isNegative = balance < 0;
    final abs = balance.abs().toStringAsFixed(2);
    return '${isNegative ? '-' : ''}\$$abs';
  }
}
```

```dart title="lib/models/transaction.dart"
import 'package:intl/intl.dart';

enum TransactionType { credit, debit }

class Transaction {
  final String id;
  final String description;
  final double amount;
  final DateTime date;
  final TransactionType type;

  const Transaction({
    required this.id,
    required this.description,
    required this.amount,
    required this.date,
    required this.type,
  });

  factory Transaction.fromJson(Map<String, dynamic> json) {
    return Transaction(
      id: json['id'] as String,
      description: json['description'] as String,
      amount: (json['amount'] as num).toDouble(),
      date: DateTime.parse(json['date'] as String),
      type: json['type'] == 'credit'
          ? TransactionType.credit
          : TransactionType.debit,
    );
  }

  String get formattedAmount {
    final prefix = type == TransactionType.credit ? '+' : '-';
    return '$prefix\$${amount.abs().toStringAsFixed(2)}';
  }

  String get formattedDate => DateFormat.yMMMd().format(date);
}
```

## 3. async/await Fundamentals

Dart uses `Future` for single asynchronous results. Every network call returns a `Future`.

```dart
// A Future represents a value that will be available later.
Future<String> fetchGreeting() async {
  await Future.delayed(const Duration(seconds: 1));
  return 'Hello from the tower!';
}

// Usage:
final greeting = await fetchGreeting();
```

Key rules:
- Mark the function `async` to use `await` inside it.
- The return type must be wrapped in `Future<T>`.
- Use `try/catch` for error handling — exactly like synchronous exceptions.
- Use `.timeout()` to prevent hanging indefinitely.

```dart
try {
  final accounts = await apiService
      .getAccounts()
      .timeout(const Duration(seconds: 10));
} on TimeoutException {
  // Show "request timed out" to the user
} on ApiException catch (e) {
  // Show "server error: ${e.message}"
} catch (e) {
  // Show generic error
}
```

## 4. Building the ApiService

Extend the service with methods for each endpoint:

```dart title="lib/services/api_service.dart"
class ApiService {
  static const _baseUrl = 'https://api.flightbank.dev/v1';
  final http.Client _client;

  ApiService({http.Client? client}) : _client = client ?? http.Client();

  /// Fetch all accounts for the current user.
  Future<List<Account>> getAccounts() async {
    final uri = Uri.parse('$_baseUrl/accounts');
    final response = await _client.get(uri);

    if (response.statusCode != 200) {
      throw ApiException(response.statusCode, response.body);
    }

    final list = jsonDecode(response.body) as List<dynamic>;
    return list
        .map((json) => Account.fromJson(json as Map<String, dynamic>))
        .toList();
  }

  /// Fetch transactions for a specific account.
  Future<List<Transaction>> getTransactions(String accountId) async {
    final uri = Uri.parse('$_baseUrl/accounts/$accountId/transactions');
    final response = await _client.get(uri);

    if (response.statusCode != 200) {
      throw ApiException(response.statusCode, response.body);
    }

    final list = jsonDecode(response.body) as List<dynamic>;
    return list
        .map((json) => Transaction.fromJson(json as Map<String, dynamic>))
        .toList();
  }

  /// Transfer money between accounts.
  Future<void> transfer({
    required String fromAccountId,
    required String toAccountId,
    required double amount,
  }) async {
    final uri = Uri.parse('$_baseUrl/transfers');
    final response = await _client.post(
      uri,
      headers: {'Content-Type': 'application/json'},
      body: jsonEncode({
        'from': fromAccountId,
        'to': toAccountId,
        'amount': amount,
      }),
    );

    if (response.statusCode != 201) {
      throw ApiException(response.statusCode, response.body);
    }
  }
}
```

## 5. FutureBuilder — Loading, Error, Data

`FutureBuilder` connects a `Future` to the widget tree. It rebuilds whenever the future's state changes: waiting, error, or data.

```dart title="lib/screens/accounts_screen.dart"
class AccountsScreen extends StatefulWidget {
  const AccountsScreen({super.key});

  @override
  State<AccountsScreen> createState() => _AccountsScreenState();
}

class _AccountsScreenState extends State<AccountsScreen> {
  final _apiService = ApiService();
  late final Future<List<Account>> _accountsFuture;

  @override
  void initState() {
    super.initState();
    _accountsFuture = _apiService.getAccounts();
  }

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('Accounts')),
      body: FutureBuilder<List<Account>>(
        future: _accountsFuture,
        builder: (context, snapshot) {
          // LOADING state — show skeleton shimmer
          if (snapshot.connectionState == ConnectionState.waiting) {
            return const _AccountsLoadingSkeleton();
          }

          // ERROR state
          if (snapshot.hasError) {
            return Center(
              child: Column(
                mainAxisSize: MainAxisSize.min,
                children: [
                  const Icon(Icons.error_outline, size: 48),
                  const SizedBox(height: 16),
                  Text('Failed to load accounts'),
                  const SizedBox(height: 8),
                  FilledButton(
                    onPressed: () {
                      setState(() {
                        _accountsFuture = _apiService.getAccounts();
                      });
                    },
                    child: const Text('Retry'),
                  ),
                ],
              ),
            );
          }

          // DATA state
          final accounts = snapshot.data!;
          if (accounts.isEmpty) {
            return const Center(child: Text('No accounts found'));
          }

          return ListView.separated(
            padding: const EdgeInsets.all(16),
            itemCount: accounts.length,
            separatorBuilder: (_, __) => const SizedBox(height: 12),
            itemBuilder: (context, index) => AccountCard(
              account: accounts[index],
            ),
          );
        },
      ),
    );
  }
}
```

A simple skeleton loader for perceived performance:

## 6. StreamBuilder Introduction

While `FutureBuilder` handles a single async result, `StreamBuilder` handles a sequence of values over time. This is useful for real-time data — like a live balance feed or push notifications.

```dart
// A stream emits multiple values over time.
Stream<double> watchBalance(String accountId) async* {
  while (true) {
    await Future.delayed(const Duration(seconds: 5));
    final accounts = await _apiService.getAccounts();
    final account = accounts.firstWhere((a) => a.id == accountId);
    yield account.balance;
  }
}
```

```dart
StreamBuilder<double>(
  stream: watchBalance('acc-001'),
  builder: (context, snapshot) {
    if (!snapshot.hasData) return const CircularProgressIndicator();
    return Text('\$${snapshot.data!.toStringAsFixed(2)}');
  },
)
```

You will use `StreamBuilder` extensively in Chapter 7 when Drift provides reactive database queries.

## 7. Error Handling Patterns

For a production app you need structured error handling. A sealed `Result` type makes success and failure explicit at the type level.

```dart title="lib/utils/result.dart"
sealed class Result<T> {
  const Result();
}

class Success<T> extends Result<T> {
  final T data;
  const Success(this.data);
}

class Failure<T> extends Result<T> {
  final String message;
  final int? statusCode;
  const Failure(this.message, {this.statusCode});
}
```

Use it in `ApiService`:

```dart
Future<Result<List<Account>>> getAccountsSafe() async {
  try {
    final accounts = await getAccounts();
    return Success(accounts);
  } on ApiException catch (e) {
    return Failure(e.message, statusCode: e.statusCode);
  } on TimeoutException {
    return Failure('Request timed out. Check your connection.');
  } catch (e) {
    return Failure('An unexpected error occurred.');
  }
}
```

Then consume it with pattern matching:

```dart
final result = await apiService.getAccountsSafe();
switch (result) {
  case Success(:final data):
    // render data
  case Failure(:final message):
    // show error with message
}
```

Implement a `getTransactionsSafe(String accountId)` method that returns `Result<List<Transaction>>`. Use the same pattern as `getAccountsSafe`.
