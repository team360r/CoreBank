<QuizQuestion
  question="Why does AnimatedSwitcher require unique keys on its children?"
  options={[
    "Keys improve rendering performance for all widgets",
    "Without keys, Flutter cannot detect that the child widget has changed and skips the animation",
    "Keys are required by the Dart compiler for all animated widgets",
    "Keys prevent the old widget from being disposed too early"
  ]}
  correctIndex={1}
  explanation="AnimatedSwitcher compares its child between builds using the widget's key and type. If you swap between two widgets of the same type (e.g., two Text widgets) without unique keys, Flutter sees them as the same widget and performs an in-place update — no transition occurs. Adding a ValueKey forces Flutter to treat them as different widgets."
/>

<QuizQuestion

  correctIndex={1}
  explanation="Interval defines a sub-range of a 0.0 to 1.0 animation timeline. By giving each list item a different Interval (e.g., item 0 at 0.0–0.4, item 1 at 0.1–0.5, item 2 at 0.2–0.6), you create a staggered cascade effect where items animate in sequence with overlap, all driven by a single AnimationController."
